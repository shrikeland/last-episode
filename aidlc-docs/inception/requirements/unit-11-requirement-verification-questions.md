### Вопрос B3: История просмотра для LLM
Как передавать историю просмотра в промпт?

A) Простой список: `[Title, status, rating, genres]` по всем тайтлам
B) Pre-summary через LLM: сначала делаем один LLM-запрос на "резюме вкусов пользователя", кешируем результат, потом используем в рекомендационном запросе
C) Гибридный подход: топ-10 высоко оцененных + топ-5 брошенных + жанровая статистика — без промежуточного LLM

[Answer]: Хотелось бы делать pre-summary, он если у человека добавлена куча тайтлов - промпт для pre-summary израсходует кучу токенов. Из-за этого придется выбирать вариант A. Хотя, возможно, ты посчитаешь иначе и при использовании каких-то дешевых моделей можно действильно выбрать вариант B
